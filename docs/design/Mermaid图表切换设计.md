# Mermaid 图表切换设计

## 目标
改进助手回复中 Mermaid 图表的呈现方式，添加每个图表的 `图表 / 代码` 切换开关，并基于现有设计系统 token 调整 Mermaid 本地控件对比度，使 UI 更清晰但不破坏现有 Markdown 渲染边界。

## 范围
- 将切换应用于助手消息中渲染的所有 Mermaid 图表。
- 默认每个 Mermaid 图表显示为可视化图表视图。
- 让用户在渲染的图表和原始 Mermaid 源代码之间切换。
- 将颜色调整限制为仅 Mermaid 图表容器及其控件。
- 保持流式输出期间的块级视图状态可恢复，不因局部重渲染而丢失。
- 保留对现有 `pre.mermaid-chart` 选择器的兼容，支持渐进迁移。

## 非目标
- 不改变全局应用图标、顶栏按钮、侧边栏控件或非 Mermaid 代码块的样式。
- 不改变普通围栏代码块的渲染方式。
- 不引入全局消息模型级别的图表/代码切换状态。
- 不在首个迭代中重做 Mermaid 主题体系或引入复杂工具栏系统。

## 用户体验
- 每个 Mermaid 图表块在右上角显示一个小型控件。
- 该控件暴露两种状态：`图表` 和 `代码`。
- 默认状态为 `图表`，显示渲染后的 Mermaid SVG。
- 切换到 `代码` 隐藏渲染的图表并显示原始 Mermaid 源代码。
- 切换回 `图表` 恢复现有渲染图表，不强制重新渲染。
- 如果 Mermaid 渲染失败，该块仍显示切换开关且源码视图保持可用。
- 失败时图表面板显示简短的本地化提示，例如“图表渲染失败，请检查语法或切换到代码查看源码”。
- 实现应使用本地化 UI 文本，中文标签应用于 zh 语言环境。
- 切换控件必须可键盘操作，并通过 `aria-pressed` 或 `aria-selected` 暴露当前状态。
- 代码面板至少支持自然文本选择与复制；结构上预留复制按钮扩展位。

## 设计原则
- 将交互限制在每个图表块本地。
- 保留当前助手消息布局和 Markdown 渲染流程，Markdown 层只负责输出语义化标记与原始内容面板。
- 使用克制的单色视觉处理，匹配当前工作区风格。
- 组件负责注入或委托渲染切换控件，避免在 Markdown 层混入交互逻辑。
- 状态由 Vue 管理，但作用域限定在 `AssistantMessage.vue` 内部，避免与流式消息数据结构耦合。
- 样式优先复用现有 CSS 变量，并在文档中明确 token 与对比度目标，而非仅描述“更深一点”。

## 渲染架构

### Markdown 层
- 更新 `frontend/src/utils/markdown.js` 中的 Mermaid 围栏处理。
- Markdown 仅输出 Mermaid 专用容器和两个“带标识的原始内容面板”，不直接输出切换按钮。
- 推荐结构：
  - 外层容器：`.mermaid-block`
  - 图表面板占位：保留 `pre.mermaid-chart` 作为兼容选择器，并增加 `data-mermaid-role="chart"`
  - 源码面板：`.mermaid-source`
- 容器上写入 `data-mermaid-block-id`，块 ID 由 Mermaid 源码哈希生成。
- 为兼容过渡，首轮实现继续保留 `pre.mermaid-chart` 类；组件层逐步从旧选择器切到 `.mermaid-block` 和 `data-mermaid-role`。
- Markdown 层不存放交互状态，只负责稳定地发射结构、源码和块 ID。

### 助手消息层
- 将 Mermaid 渲染保留在 `frontend/src/components/chat/AssistantMessage.vue` 内。
- 由组件在每个 `.mermaid-block` 内注入或委托渲染切换控件，而不是由 Markdown 拼接工具栏 HTML。
- 渲染目标缩小为每个容器内的图表面板，占位面板只承担 SVG 容器角色。
- 每个 Mermaid 源代码渲染一次，将 SVG 保留在原位，仅在图表和代码面板之间切换可见性。
- 组件内部维护块 ID 到视图状态的映射，并在 DOM 重建后根据 `data-mermaid-block-id` 恢复选中的面板。
- Mermaid 渲染期间必须捕获异常，保证单块失败不会破坏整条消息或页面其他图表。

### 视图状态
- 状态由 `AssistantMessage.vue` 通过本地响应式对象管理，而不是写回消息对象或全局 store。
- 键为 `data-mermaid-block-id`，值为 `chart` / `code` / `retry-pending` 等块级状态。
- DOM 容器同步持有一份 `data-mermaid-block-id` 和当前 `data-view`，用于重渲染后的状态恢复与调试。
- 当同一块在流式过程中被重新输出时，只要源码哈希不变，就恢复用户上次选择的视图。
- 当源码变化导致哈希变化时，视为新块，默认回到图表视图并重新渲染。

## 交互模型
- 点击 `图表` 将块设置为图表模式。
- 点击 `代码` 将块设置为代码模式。
- 活跃状态样式应清晰指示哪个模式被选中。
- 隐藏面板应保持挂载，因此切换不会破坏渲染结果或源码内容。
- 控件可通过 `Tab` 聚焦，并可通过 `Enter` / `Space` 激活。
- 若采用按钮组实现，使用 `aria-pressed`；若采用 tabs 实现，使用 `aria-selected`、`role="tablist"` 和对应面板关联。
- 代码面板不阻断原生文本选择；如后续添加复制按钮，应复用同一工具栏扩展位。

## 流式行为
- 新插入的 Mermaid 块默认为图表模式。
- 流式更新可能引入新的 Mermaid 块；观察者应仍能发现并渲染它们。
- 已渲染的现有 Mermaid 块不应仅因用户切换视图而重新渲染。
- 如果流式响应在容器存在后完成，Mermaid 渲染应在图表面板就绪时进行一次。
- 观察者除监听新增节点外，还应监听 `.mermaid-source` 文本内容变化，以识别流式补全后的源码更新。
- 渲染失败时将块标记为“待重试”；重试可由短延时定时器触发，并在流式结束事件或 `message.rendered` 变为完成时再做一次最终尝试。

## 视觉设计
- 将颜色和对比度更改限制为 Mermaid 容器和控件。
- 优先复用现有 token：`--surface-soft`、`--surface`、`--border-soft`、`--border-strong`、`--text-main`、`--text-muted`、`--focus-ring`。
- 若必须新增 Mermaid 专用类，应在文档和样式中给出明确颜色值或对比度目标，至少保证控件文本与背景满足可读性要求。
- 工具栏、活跃切换状态、边框和悬停状态使用比当前 Mermaid 本地样式更明确的灰度层次，但不覆盖全局按钮语言。
- 保持图表表面其余部分干净、编辑化而非装饰性。

## 错误处理
- Mermaid 渲染失败不得留下空块。
- 在失败模式下，保持工具栏可见并允许切换到 `代码`。
- 图表面板显示紧凑的本地化降级消息，但源码访问是重要的降级方案。
- 失败块保留源码文本，且异常需被捕获并记录在局部范围内，不能中断整个消息渲染。
- 失败后进入 `retry-pending` 状态，直到源码变化或最后一次重试结束。

## 测试策略

### Markdown 测试
- 扩展 `frontend/src/utils/__tests__/markdown.spec.js` 以验证 Mermaid 围栏渲染专用容器结构。
- 断言输出包含 `.mermaid-block`、`pre.mermaid-chart`、`.mermaid-source` 和稳定的块 ID 标记。
- 断言普通围栏代码块仍走原有路径。

### 组件测试
- 为 `frontend/src/components/chat/AssistantMessage.vue` 添加或扩展测试以涵盖：
  - 默认图表模式
  - 切换到代码模式
  - 切换回图表模式
  - 基于相同块 ID 的视图状态恢复
  - Mermaid 渲染失败保留源码访问
  - 失败后的待重试与流式结束后的最终重试
  - 键盘操作与 ARIA 状态输出

### 手动 / E2E 清单
- 流式输出过程中首次出现 Mermaid 块时，默认显示图表，且切换控件不会导致流式文本闪烁或重置。
- 在流式尚未结束时切到 `代码`，后续增量内容到达后应保持当前块仍处于代码视图。
- 同一条消息中多个 Mermaid 块可分别切换，互不影响。
- 制造 Mermaid 语法错误时，页面不报致命错，图表面板显示本地化错误提示，源码仍可查看。
- 使用键盘完成聚焦、切换，并验证 `aria-pressed` 或 `aria-selected` 随状态变化。
- 代码面板可以直接拖选复制；若复制按钮落地，还需验证其成功提示与无障碍标签。

### 回归预期
- 现有非 Mermaid 围栏代码块渲染保持不变。
- 现有助手消息渲染仍支持流式输出和追问操作。
- 旧的 `pre.mermaid-chart` 样式和选择器在过渡期内仍可工作。

## 风险
- 如果 Mermaid HTML 结构过于紧密耦合到当前选择器，渲染钩子可能错过新块。
- 如果块 ID 哈希策略未稳定规范化换行或空白，重渲染时可能错误地将同一块识别为新块。
- 如果图表和代码面板通过脆弱的 DOM 遍历切换，流式更新可能意外重置活跃视图。
- 如果观察者范围过大或重试策略过于频繁，可能引入重复渲染和性能抖动。
- 如果控件对比度过于激进地增加，工具栏可能在视觉上压倒图表。

## 验证
- 助手消息中所有 Mermaid 图表显示 `图表 / 代码` 切换。
- 默认视图为渲染的图表。
- 用户可以在不丢失渲染 SVG 的情况下切换到源代码并返回。
- Mermaid 失败仍留下可用的源码视图。
- Mermaid 块在局部重渲染后能按块 ID 恢复先前视图。
- 仅 Mermaid 本地控件变暗；无关应用图标保持不变。
- 切换控件满足键盘操作与 ARIA 暴露要求。

# 可折叠侧边栏题库设计

**日期：** 2026-05-13

## 目标

调整聊天页侧边栏，使示例问题更贴近原型意图：

- 保留当前六个顶层业务场景名称
- 将侧边栏变为可折叠的题库
- 在每个模块下预置两个标准问题
- 通过现有语言系统支持中英文热切换
- 在提交开始时对被点击的问题显示轻量内联加载反馈

## 范围

本设计仅涵盖聊天页的前端侧边栏题库体验。

包含：

- 侧边栏渲染和展开行为
- 中文和英文的本地化题库数据更新
- 问题被点击时的直接发送行为
- 可折叠模块交互的测试更新

排除：

- 后端变更
- 模型调用逻辑
- 聊天消息渲染变更
- 会话管理变更
- 动态问题生成

## 问题陈述

当前侧边栏不再匹配期望的交互模型。

当前行为：

- 显示六个直接发送的问题按钮
- 每个业务场景仅暴露一个问题
- 没有可折叠的分组

期望行为：

- 六个场景作为模块保持可见
- 每个模块默认折叠
- 展开模块显示两个预定义的标准问题
- 点击问题立即发送
- 切换语言同时更新模块标题和问题文本
- 当前展开的模块在语言切换时保持展开状态
- 被点击的问题显示小型加载旋转器或柔和的加载状态

## 设计决策

### 1. 保留当前六个场景名称

现有六个场景名称已与当前代码和最近的侧边栏结构对齐。此变更应深化该结构，而非重命名或重新分类。

### 2. 使用轻量手风琴替代流程卡片

侧边栏应表现得像一个标准题库，而非工作流解释器。

每个模块显示：

- `title`（标题）
- 展开/折叠图标

每个展开的模块显示：

- 恰好两个直接发送的问题按钮

此版本中没有摘要、工具标签或步骤预览。

### 3. 默认状态下全部折叠

页面加载时，所有六个模块均折叠。

这使侧边栏保持紧凑，让模块感觉像可浏览的库，而非已打开的清单。

### 4. 单模块展开行为

一次仅应展开一个模块。

这使侧边栏在桌面端和移动端保持整洁，避免一长串堆叠的开放问题。

### 5. 问题点击保持现有直接提交路径

点击问题应继续复用现有侧边栏提交路径：

- 关闭移动端侧边栏
- 调用 `submitPrompt(question)`

这保留了当前验证和模型设置门控，而不引入第二条代码路径。

### 6. 被点击的问题显示本地加载反馈

当用户点击问题时，该特定按钮应在请求被交接期间显示轻量等待状态。

可接受的视觉处理：

- 小型内联旋转器
- 被点击按钮上柔和的灰色加载覆盖

其目的是确认点击被接收，而不强制整个手风琴视觉锁定。

### 7. 语言切换保留结构化 UI 状态

如果用户展开了一个模块然后切换语言，侧边栏应通过稳定的模块标识保持展开的模块，而非全部折叠。

示例：

- 用户在中文下展开第一个模块
- 用户切换到英文
- 对应模块保持展开
- 其标题和两个问题立即更新为英文

这必须仅通过响应式状态更新实现，不重新加载页面，也不重置手风琴状态。

## 数据模型

本地化侧边栏数据应保留在 `frontend/src/constants/sidebarFlows.js` 中，并继续通过 `frontend/src/i18n/messages.js` 向 `promptModules` 输送。

每个模块应遵循以下结构：

```js
{
  title: "...",
  questions: ["...", "..."]
}
```

规则：

- 所有六个模块必须在两种语言中都存在
- 每个模块必须提供恰好两个标准问题
- 语言切换必须替换完整的本地化模块集

## 组件设计

### `frontend/src/components/layout/ExampleSidebar.vue`

变更后的职责：

- 将六个模块渲染为可折叠分区
- 按稳定的模块 ID 在局部追踪当前展开的模块
- 追踪当前哪个问题正在等待提交
- 默认不展开模块
- 当模块头部被点击时展开或折叠该模块
- 为展开的模块渲染两个问题按钮
- 当问题被点击时发射 `submit-prompt`

交互细节：

- 模块头部点击从不发送提示词
- 问题按钮点击始终发送提示词
- 当 `isSending` 为 `true` 时问题按钮被禁用
- 被点击的问题在点击后立即显示本地加载状态
- 发送期间展开/折叠仍可用
- 语言驱动的字典变化在原位更新标题和问题文本，不重置展开的模块

### `frontend/src/views/ChatView.vue`

除保持现有直接提交处理函数与侧边栏的连接外，不需要结构性变更。

当前 `handleSubmitSidebarPrompt(prompt)` 行为已是正确的执行路径。

## 交互流程

### 桌面端和移动端

1. 用户打开聊天页。
2. 所有六个侧边栏模块均为折叠状态。
3. 用户点击一个模块标题。
4. 该模块展开，任何之前打开的模块折叠。
5. 用户点击两个标准问题之一。
6. 被点击的问题显示小型本地加载图标。
7. 问题通过现有聊天流程立即提交。

在移动端，侧边栏在提交前关闭，与当前行为一致。

如果用户在模块展开时切换语言，同一模块保持打开，其可见标题和问题立即以新语言刷新。

## 错误处理

- 如果 `isSending` 为 `true`，问题按钮被禁用。
- 被点击的问题应在全局发送状态禁用列表之前进入其等待视觉状态。
- 如果模型设置不完整，侧边栏提交仍触发现有的设置门控。
- 如果提交失败，现有请求错误行为保持不变。
- 如果语言在模块打开时变化，展开状态不应破坏；同一模块应对新字典保持打开。

## 测试策略

更新测试以证明：

- 模块从 `promptModules` 渲染
- 所有模块默认折叠
- 点击模块头部显示两个问题按钮
- 打开一个模块折叠之前打开的模块
- 点击问题以选中文本发射 `submit-prompt`
- 点击问题在该按钮上显示本地等待状态
- 发送中问题按钮被禁用
- 本地化字典可以渲染不同的模块/问题文本
- 切换语言保留展开的模块并在不移除它的情况下刷新其可见文本

# 模型设置本地设计

## 目标
在聊天顶栏添加"设置"按钮，让用户可以在前端配置 `baseUrl`、`apiKey` 和 `model`。配置必须存储在本地浏览器中，并立即应用于下一条聊天消息。

## 范围
- 聊天顶栏中的纯前端设置 UI。
- 将设置持久化到 `localStorage`。
- 为当前页会话保留已保存设置的内存副本。
- 随每次聊天请求发送已保存的设置。
- 为模型设置添加专用的连接测试操作。
- 要求在发送任何聊天消息之前，用户必须提供完整的模型配置。

## 非目标
- 不做后端数据库存储。
- 不做跨设备账号级同步。
- 不做服务端密钥保管。
- 不在主用户流程中提供系统默认模型路径。

## 用户体验
- 在现有顶栏操作按钮旁添加"设置"按钮。
- 打开一个小型模态框或抽屉，包含三个字段：
  - `Base URL（基础 URL）`
  - `API Key（API 密钥）`
  - `Model（模型）`
- 包含"测试连接"、"保存"、"取消"和"重置"。
- "保存"立即应用于下一条请求。
- "重置"清除浏览器存储的设置，将应用恢复为未配置状态。
- 当用户输入的不完整自定义配置缺少任何必填字段时，以内联方式显示验证错误。
- 即使没有配置模型，也允许用户打开聊天页并在输入框中输入。
- 如果用户在没有保存模型配置的情况下点击发送：
  - 阻止请求；
  - 显示"请先打开设置"风格的提示；
  - 自动打开设置面板；
  - 不创建占位的助手消息。

## 数据模型
在 `localStorage` 中存储一个浏览器范围的对象：
```json
{
  "baseUrl": "https://api.openai.com",
  "apiKey": "sk-...",
  "model": "gpt-4o-mini"
}
```

在页面初始化后，在响应式前端状态中保持相同结构。聊天流应在成功保存后使用当前内存中的状态，而不是每次请求前重新读取 `localStorage`。

## 请求流程
1. 聊天视图在初始化时从 `localStorage` 加载设置一次。
2. 用户在模态框中编辑设置。
3. "测试连接"将候选设置发送到专用后端端点。
4. "保存"同时更新当前页面状态和 `localStorage`。
5. 下一次聊天提交从当前页面状态中包含已保存的设置。
6. 后端从请求负载中解析聊天模型。

## 连接测试
- 添加专用后端端点，如 `POST /api/model-config/test`。
- 请求体：
  - `baseUrl`
  - `apiKey`
  - `model`
- 职责：
  - 验证目标端点是否可达；
  - 验证认证是否成功；
  - 避免创建聊天历史或调用主聊天工作流。
- 尽可能返回明确的失败类别，如：
  - 配置不完整；
  - URL 无效；
  - 认证失败；
  - 超时；
  - 上游不可达。

## 后端变更
- 使用模型配置字段扩展 `ChatRequest`。
- 为连接测试添加专用的 DTO 和控制器端点。
- 更新聊天处理，从请求负载中解析模型设置，而非当前的进程级默认路径。
- 拒绝不完整或缺失模型配置的聊天请求，而非以无模型降级流程继续。
- 避免记录原始 API 密钥。

## 错误处理
- 如果用户保存了无效的 URL 或不完整的自定义配置，阻止保存并显示验证反馈。
- 如果用户以无效输入点击"测试连接"，阻止请求并显示验证反馈。
- 如果用户在没有保存配置的情况下点击发送，显示设置提示并打开面板。
- 如果聊天 API 以已保存的设置失败，返回明确的请求错误，而非静默降级到不同的聊天模式。
- 如果连接测试失败，保持草稿设置可见，以便用户修正。

## 安全说明
- `apiKey` 在此阶段有意存储在 `localStorage` 中，因为需求是浏览器本地持久化且立即可复用。
- 这对于当前的单浏览器工作流是可接受的，但不等同于安全的密钥存储，如果应用超出本地或演示用途，应重新审视。

## 测试
- 前端：
  - 渲染新的设置入口点。
  - 打开、取消、保存和重置设置面板。
  - 将设置保存到页面状态和 `localStorage`。
  - 页面加载时从 `localStorage` 恢复设置。
  - 当没有保存模型配置时阻止发送并打开设置面板。
  - 在下一次聊天请求中包含已保存的设置。
  - 处理连接测试的成功和失败状态。
- 后端：
  - 在 `ChatRequest` 上接受模型设置。
  - 拒绝缺失或不完整模型设置的聊天请求。
  - 支持专用连接测试端点。
  - 对认证失败、无效输入和上游连接问题返回明确的失败响应。

## 验收标准
- 用户可以从聊天页配置模型设置。
- 设置在同一浏览器的刷新后持久化。
- 新保存的设置在下一条消息上生效，无需重启应用。
- 用户必须有完整的模型配置后才能发送任何聊天消息。
- 在没有保存配置的情况下点击发送会打开设置面板并显示清晰提示。
- 用户可以在保存前从设置面板验证连接和认证。

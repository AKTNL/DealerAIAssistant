# 黑白编辑风格 UI 设计

## 目标
将前端从柔和的绿色玻璃态风格转换为以白色为主、黑/灰色为辅的编辑风格。

## 设计原则
- 白色背景，黑色文字，仅使用灰色层级。
- 不引入新的强调色。
- 通过间距、字体排版、边框和对比度来构建结构。
- 保持 UI 锐利、克制、专业，而非厚重或仪表盘风格。

## 视觉体系
- 背景：页面画布使用 `#f8f8f6`。
- 主表面：`#ffffff`。
- 主文字：`#111111`。
- 次要文字：`#5f5f5f`。
- 辅助文字：`#8a8a8a`。
- 边框：`#d9d9d4`。
- 分割线：`#e8e8e3`。
- 阴影：尽量少用；优先使用细边框。
- 圆角：主要为 `8px` 到 `12px`；避免过度使用药丸形状。
- 按钮：扁平、单色、无渐变、无发光、无玻璃效果。

## 页面行为

### 登录页
- 将登录页视为封面页。
- 让标题在视觉上占主导地位。
- 保留 Logo 但使其处于次要位置。
- 将登录表单隔离为一个窄而干净的区块。
- 减少居中浮动卡片的感觉；更多依赖字体排版和构图。
- 使用白色表面、细边框和强烈的间距，而非装饰性背景效果。

### 聊天页
- 将聊天页视为工作编辑台。
- 保留左侧边栏，但减少装饰。
- 保持顶栏纤细且功能性强。
- 通过布局和边框处理来区分发送者状态，而非颜色。
- 助手消息应更中性，以内容为导向。
- 用户消息可使用略深的灰色表面或更强的边框，但仍保持在灰度范围内。
- 避免基于颜色的消息区分。
- 让输入区域感觉像一个干净的编辑面板。

## 组件规则
- 主按钮：黑色背景，白色文字。
- 次要按钮：白色背景，黑色文字，灰色边框。
- 禁用状态：仅降低对比度。
- 输入框：白色背景，细灰色边框，聚焦时呈微妙灰色。
- 侧边栏：比主画布略偏冷的米白色。
- 消息卡片：仅白色或非常浅的灰色。
- 头像：单色、简化、比当前更低的视觉权重。
- 标题：浓重的黑色。
- 正文：深灰色。
- 辅助文字：中灰色。
- 状态文字：仅灰度。

## 实施范围
- 更新 `frontend/src/style.css` 中的全局 token。
- 保持 `LoginView.vue` 和 `ChatView.vue` 中现有的页面结构。
- 在改变布局结构之前，先重新设计共享 UI 组件，如顶部导航、侧边栏、聊天消息和输入区域。
- 从主用户流中移除可见的绿色强调色、渐变和玻璃态效果。

## 非目标
- 不做暗色模式重新设计。
- 不做新的品牌色系统。
- 不做超出此视觉转变所需的布局重构。

## 风险
- 过多的留白可能使密集的聊天内容显得空旷。
- 过多的边框可能使 UI 显得脆弱。
- 如果灰色值过于接近，层级会崩塌。

## 验证
- 登录页呈现为克制的封面页。
- 聊天页感觉像专业的分析工作空间。
- 可见 UI 中不残留绿色或其他强调色。
- UI 在桌面端和移动端保持可读。
- 按钮层级在不使用颜色的情况下仍然明显。
- 用户消息和助手消息一眼就能区分。

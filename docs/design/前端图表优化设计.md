# 前端图表优化 — 设计方案

## 背景

当前回复中有两套图表渲染体系，但定位与效果差异明显：

| 体系 | 方案 | 数据来源 | 图表类型 | 视觉效果与交互 |
|------|------|---------|---------|-------------|
| 数值对比图表 | **ECharts**（`AnalysisChart.vue`） | 前端从 HTML `<table>` 解析结构化数据 | 柱状图、热力图、雷达图 | 专业数据可视化，tooltip 交互，色阶编码，排序/筛选控件 |
| AI 生成图表 | **Mermaid**（`AssistantMessage.vue`） | 后端在 Markdown 中输出 ` ```mermaid ` 代码块 | xychart 柱状图、pie 饼图 | 静态 SVG，零交互，CSS 后处理着色，布局控制力弱 |

**核心矛盾**：柱状图/饼图/数据对比恰好是 Mermaid 最弱的环节、ECharts 最强的环节。后端 6 个分析场景（目标达成、商机漏斗、销售跟进、活动效果、线索来源、门店对标）生成的 Mermaid xychart 和 pie 图表，用 ECharts 展示效果会显著更好。

## 设计目标

1. **数值对比类图表全面迁移到 ECharts**：柱状图、饼图从 Mermaid 迁移到 ECharts
2. **Mermaid 回归其核心定位**：仅用于流程图、时序图、甘特图等结构性图表（如 LLM 自由生成的架构图）
3. **向后兼容**：不影响已有的 `AnalysisChart.vue`（从 HTML Table 解析数据的通道），新增从 chart-json fence 块解析数据的通道
4. **渐进式迁移**：先覆盖后端 6 个分析场景的固定图表，保留 Mermaid 渲染能力以兼容未来 LLM 自由生成的其他图表
5. **健壮性优先**：JSON 解析失败降级为普通代码块、空数据展示空状态而非空图、流式输出中展示 Loading 而非抖动

## 一、现状分析

### 1.1 后端图表生成（ReportRenderer.java）

后端在构建分析回复时，通过以下方法生成 Mermaid 图表代码块：

- **`mermaidXyChart()`**：生成 `xychart-beta` 柱状图（用于 6 个场景的数值对比）
- **`mermaidPie()`**：生成 `pie` 饼图（用于商机漏斗、线索来源场景）
- **`fallbackBars()`**：生成纯文本 ASCII 柱状图（图表被抑制时降级）

这些方法被 `RuleBasedAnalyticsService.java` 的 6 个场景方法调用，输出为 Markdown 中的 ` ```mermaid ` 代码块。

### 1.2 前端 Mermaid 渲染链路

```
后端 Markdown → markdown.js fence 规则检测 "mermaid"
  → 生成 <div class="mermaid-block"> → AssistantMessage.vue
  → mermaid.render() → SVG → CSS 后处理着色
```

Mermaid xychart 的样式完全依赖 `style.css:2579-2646` 的 CSS 后处理：
- 柱子颜色用 `nth-of-type` 选择器轮询赋值（蓝、橙、绿、紫、金、青）
- 标题、坐标轴颜色通过 `!important` 覆盖 SVG 内联样式
- 零交互能力，tooltip、hover 高亮、缩放均不支持

### 1.3 前端 ECharts 渲染链路（对照）

```
后端 Markdown（HTML table） → analysisCharts.js 解析 <table>
  → 构建 chart model（type/rows/metrics） → AnalysisChart.vue
  → echarts.init() → Canvas 渲染 → tooltip + 交互控件
```

ECharts 的优势：
- **有 tooltip**：鼠标悬停查看精确数值
- **色阶编码**：最高蓝色 `#2f7da8`，最低红色 `#a43f3f`，中间灰色 `#6f879d`
- **交互控件**：排序、筛选、Top-N 过滤
- **专业布局**：网格系统、自适应宽度、坐标轴智能截断

## 二、前后端 JSON 协议

### 2.1 柱状图（Bar）

```json
{
  "type": "bar",
  "title": "各门店达成率对比",
  "metric": "达成率",
  "metricType": "percentage",
  "categories": ["门店A", "门店B", "门店C"],
  "values": [85.2, 92.1, 45.2],
  "averageLine": 74.2
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 固定值 `"bar"` |
| `title` | string | 是 | 图表标题 |
| `metric` | string | 是 | 指标名称，用于 tooltip 和坐标轴标签 |
| `metricType` | string | 否 | `"percentage"` 或 `"absolute"`，默认 `"absolute"`。决定色阶策略（见 4.2） |
| `categories` | string[] | 是 | Y 轴标签（门店名/活动名等），**后端负责截断至 ≤10 字符** |
| `values` | number[] | 是 | 与 categories 一一对应的数值 |
| `averageLine` | number | 否 | 平均值参考线，存在时渲染 ECharts markLine |

### 2.2 饼图（Pie）

```json
{
  "type": "pie",
  "title": "线索来源分布",
  "slices": [
    { "name": "线上广告", "value": 45 },
    { "name": "线下活动", "value": 30 },
    { "name": "老客户推荐", "value": 15 },
    { "name": "其他", "value": 10 }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 固定值 `"pie"` |
| `title` | string | 是 | 图表标题 |
| `slices` | {name, value}[] | 是 | 每个扇区的名称和数值。**后端限制最多 6 项（Top 5 + 其他）** |

### 2.3 空数据 / 无数据（Empty）

当后端 6 个场景在极端情况下算不出数据，或 values 全为 0/null 时：

```json
{
  "type": "bar",
  "title": "各门店达成率对比",
  "metric": "达成率",
  "metricType": "percentage",
  "categories": [],
  "values": [],
  "emptyMessage": "当前范围内暂无有效的门店达成率数据"
}
```

或 pie 空数据：

```json
{
  "type": "pie",
  "title": "线索来源分布",
  "slices": [],
  "emptyMessage": "当前范围内暂无有效的线索来源数据"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `emptyMessage` | string | 否 | 仅当 `categories/values/slices` 为空时存在。前端据此展示空状态而非空图。后端应根据 locale 返回中/英文 |

**后端判定规则**：
- `values` 全部为 `null` → 输出空数组 + `emptyMessage`
- `values` 全部为 `0` → 输出空数组 + `emptyMessage`（全零无对比意义）
- 去重后有效 label 数 < 2 → 输出空数组 + `emptyMessage`

### 2.4 Markdown 输出格式

后端在流式 Markdown 中输出 `chart-json` 围栏代码块：

````markdown
## 数据支撑

<table>...全量数据表格...</table>

```chart-json
{"type":"bar","title":"各门店达成率对比","metric":"达成率","metricType":"percentage","categories":["门店A","门店B","门店C"],"values":[85.2,92.1,45.2],"averageLine":74.2}
```
````

**关键约束**：
- JSON 文本必须是 **单行压缩格式**（Minified JSON），由 Jackson/Gson 的标准序列化器生成，确保是 Strict JSON（双引号、合法转义、无 trailing comma）
- 流式输出过程中，前端仅在检测到闭合的 ` ``` ` 后才触发解析（见 3.4）

## 三、后端实现要点

### 3.1 JSON 序列化

**强制使用 Jackson（项目已有依赖）构建 JSON**，严禁手动拼接字符串。

```java
// ReportRenderer.java 新增方法

private static final ObjectMapper JSON_MAPPER = new ObjectMapper();

public String chartJsonBar(String title, String metric, String metricType,
        List<String> categories, List<Double> values, Double averageLine) {

    Map<String, Object> model = new LinkedHashMap<>();
    model.put("type", "bar");
    model.put("title", title);
    model.put("metric", metric);
    if (metricType != null) {
        model.put("metricType", metricType);
    }
    model.put("categories", categories != null ? categories : List.of());
    model.put("values", values != null ? values : List.of());

    if (averageLine != null) {
        model.put("averageLine", averageLine);
    }

    // 空数据判定
    if (isEmptyOrAllZero(values) || distinctCount(categories) < 2) {
        model.put("categories", List.of());
        model.put("values", List.of());
        model.put("emptyMessage", resolveEmptyMessage(title, metric));
    }

    try {
        String json = JSON_MAPPER.writeValueAsString(model);
        return "```chart-json\n" + json + "\n```";
    } catch (JsonProcessingException e) {
        // 兜底：返回 chart-empty fence 块，绝不输出损坏的 JSON
        return "```chart-empty\nreason: SERIALIZATION_ERROR\ntitle: Chart unavailable\nbody: Failed to serialize chart data.\n```";
    }
}
```

**度量类型设置规则**：

| 数据特征 | `metricType` |
|---------|-------------|
| 达成率、完成率、转化率等百分比指标 | `"percentage"` |
| 销售额、任务数、线索量等绝对数值 | `"absolute"`（或省略，即默认） |

### 3.2 改动范围

| 文件 | 改动 |
|------|------|
| `ReportRenderer.java` | 新增 `chartJsonBar()`、`chartJsonPie()`；新增 `isEmptyOrAllZero()`、`resolveEmptyMessage()` 辅助方法。保留原有 Mermaid 方法（不删除）。使用 Jackson `ObjectMapper` 序列化 |
| `RuleBasedAnalyticsService.java` | 6 个场景方法：将 `mermaidXyChart()` / `mermaidPie()` 替换为 `chartJsonBar()` / `chartJsonPie()`；传入正确的 `metricType` |
| `RuleBasedAnalyticsServiceTest.java` | 更新测试断言：改为校验 JSON 结构字段；新增空数据/全零场景的 `emptyMessage` 测试 |
| `ReportRendererTest.java` | 新增 JSON 输出的单元测试；新增 Jackson 序列化失败兜底测试 |

## 四、前端实现要点

### 4.1 markdown.js — 识别 chart-json 围栏

在 `frontend/src/utils/markdown.js` 的 `fence` 规则中新增对 `chart-json` 的检测。

**核心逻辑**：

```javascript
// markdown.js fence 规则中
if (token.info === "chart-json") {
  // 生成占位 DOM，JSON 原文存入 data 属性，由 Vue 组件接管
  const escaped = escapeHtml(token.content);
  return `<div class="chart-json-block" data-chart-json="${escaped}"></div>`;
}
```

**防御性设计**：
- `data-chart-json` 中存储的是后端原始 JSON 字符串（HTML 转义后）
- 不在此层做 JSON.parse — 解析工作交给 Vue 组件层，以便 try-catch 和降级

### 4.2 懒加载 MermaidChartAdapter.vue

在 `AssistantMessage.vue` 中使用 Vue 3 的 `defineAsyncComponent` 进行懒加载。只有当 markdown 解析结果中包含 `.chart-json-block` 元素时，才触发 ECharts 相关 Bundle 的加载。

```javascript
// AssistantMessage.vue <script setup>
import { defineAsyncComponent } from "vue";

const MermaidChartAdapter = defineAsyncComponent(() =>
  import("./MermaidChartAdapter.vue")
);
```

**懒加载流程**：

```
message.html 更新 → MutationObserver / watch 检测 DOM 中 .chart-json-block
  → 存在 → 触发 defineAsyncComponent 加载 → 渲染 <MermaidChartAdapter>
  → 不存在 → 不加载 ECharts 相关代码
```

**收益**：绝大多数不含图表的回复不会触发 ECharts 代码加载，减少首屏和每次消息渲染的 JS 开销。

### 4.3 流式输出中的 Loading 状态

**约束**：流式输出过程中，chart-json 围栏内容是不完整的。只有当 markdown 解析器检测到完整的闭合 ` ``` ` 之后，才会产出 fence token。在此之前，`token.content` 不存在。

**因此 markdown.js 层面的 fence 回调天然保证了闭合性**，不需要额外的前端状态管理。

**Loading 展示**：

在流式输出期间，markdown 渲染器会展示一个通用的「图表生成中...」Loading 占位（markdown.js 可在检测到 `chart-json` 开始标记但尚未闭合时输出一个占位 div）。实际闭合后，替换为 `.chart-json-block` 并由 `MermaidChartAdapter` 接管渲染。

```
流式输出中：
  ```chart-json        → <div class="chart-json-loading">图表生成中...</div>
  (内容陆续到达...)
  ```                  → <div class="chart-json-block" data-chart-json="...">
                          → MermaidChartAdapter 初始化 ECharts
```

#### 4.4.2 JSON 解析与降级策略

```javascript
// 组件内部
function parseChartJson(raw) {
  try {
    return JSON.parse(raw);
  } catch (e) {
    console.warn("chart-json parse failed, rendering as code block", e);
    return null; // 返回 null → 模板降级为 <pre><code> 代码块
  }

```

**降级行为**：解析失败时，渲染一个标准的 `<pre><code>` Markdown 代码块，包含原始 JSON 文本。**页面不白屏、不崩溃**。

#### 4.4.3 空数据状态

当 `model.values.length === 0` 或 `model.slices.length === 0` 时，**不渲染只有坐标轴的空图**，改为展示空状态：

```html
<div class="chart-empty-state">
  <p class="chart-empty-title">{{ model.title }}</p>
  <p class="chart-empty-body">{{ model.emptyMessage || '暂无可用数据' }}</p>
</div>
```

复用已有的 `analysis-empty-chart` 样式（`style.css:2469-2491`）。

#### 4.4.4 色阶策略：固定阈值 vs 相对阈值

**核心原则**：根据 `metricType` 选择着色策略，使用 ECharts 回调函数（`series.itemStyle.color` 支持 Function）逐个数据点计算颜色。

```javascript
function buildBarOption(model) {
  const categories = model.categories;
  const values = model.values;
  const isPercentage = model.metricType === "percentage";

  // 计算动态阈值（仅在 absolute 模式下使用）
  const maxVal = Math.max(...values);
  const minVal = Math.min(...values);
  const range = maxVal - minVal || 1;

  return {
    // ...grid, tooltip, xAxis, yAxis...
    series: [{
      type: "bar",
      barMaxWidth: 28,
      data: values.map((v, i) => ({
        name: categories[i],
        value: v,
        itemStyle: {
          color: colorForValue(v, isPercentage, maxVal, minVal, range)
        }
      }))
    }]
  };
}

function colorForValue(value, isPercentage, maxVal, minVal, range) {
  if (isPercentage) {
    // 固定阈值：达成率/转化率等百分比指标
    if (value >= 80) return "#2f7da8";   // 良好：蓝色
    if (value >= 60) return "#6f879d";   // 一般：灰色
    return "#a43f3f";                     // 预警：红色
  }

  // 相对阈值：绝对数值（销售额、任务数等），动态色阶
  const ratio = (value - minVal) / range; // 0 到 1

  if (ratio >= 0.7) return "#2f7da8";     // 高位区：蓝色
  if (ratio >= 0.3) return "#6f879d";     // 中间区：灰色
  return "#a43f3f";                       // 低位区：红色
}
```

**回调函数方式**：ECharts 的 `itemStyle.color` 支持 function 类型的值，会在每个数据点渲染时调用，天然实现逐点着色：

```javascript
itemStyle: {
  color: function (params) {
    return colorForValue(params.value, isPercentage, maxVal, minVal, range);
  }
}
```

#### 4.4.5 饼图防标签重叠

**双层防线**：

**防线 1 — 后端限制**：`ReportRenderer.chartJsonPie()` 最多输出 6 个 slice（Top 5 + "其他"合并）。合并逻辑：

```java
// 按 value 降序排列，前 5 保留，其余合并为"其他"
if (slices.size() > 6) {
    // 取前 5
    List<Slice> top5 = sorted.subList(0, 5);
    double otherValue = sorted.subList(5, sorted.size()).stream()
        .mapToDouble(Slice::value).sum();
    top5.add(new Slice("其他", otherValue));
    slices = top5;
}
```

**防线 2 — ECharts 配置**：环形图（Donut）+ 标签碰撞避免：

```javascript
series: [{
  type: "pie",
  radius: ["45%", "70%"],           // 环形图，中间留白
  center: ["50%", "55%"],           // 垂直居中偏上
  avoidLabelOverlap: true,          // ECharts 内置标签防重叠
  label: {
    minMargin: 5,                   // 标签间最小间距（px）
    edgeDistance: 10,               // 标签距边缘最小距离
    formatter: "{b}: {d}%"          // 名称 + 百分比
  },
  labelLine: {
    length: 16,
    length2: 24
  },
  emphasis: {
    label: {
      fontSize: 14,
      fontWeight: "bold"
    }
  }
}]
```

**选择环形图而非纯饼图的原因**：
- 标签文本居中显示在环形空白处，大幅减少重叠
- 数据项较多时视觉上更整洁
- ECharts 原生支持，无需额外开发

#### 4.4.6 容器大小自适应

```javascript
// onMounted
let resizeObserver = null;

onMounted(async () => {
  await nextTick();
  if (!chartEl.value) return;

  chart = init(chartEl.value);
  chart.setOption(buildOption());

  resizeObserver = new ResizeObserver(() => {
    chart?.resize();
  });
  // 监听自身容器，同时向上传递到父容器尺寸变化
  resizeObserver.observe(chartEl.value);

  // 也监听窗口 resize（ResizeObserver 已覆盖大多数场景，但窗口 resize 是兜底）
  window.addEventListener("resize", handleWindowResize);
});

onBeforeUnmount(() => {
  resizeObserver?.disconnect();
  window.removeEventListener("resize", handleWindowResize);
  chart?.dispose();
});

function handleWindowResize() {
  chart?.resize();
}
```

#### 4.4.7 非 bar/pie 类型降级

```javascript
function renderChart(model) {
  if (model.type === "bar") {
    chart.setOption(buildBarOption(model));
  } else if (model.type === "pie") {
    chart.setOption(buildPieOption(model));
  } else {
    // 未知类型：不做任何渲染，控制台警告
    console.warn("Unknown chart-json type:", model.type);
  }
}
```

## 六、风险与应对

| 风险 | 应对 |
|------|------|
| Jackson 序列化失败 | catch 后输出 `chart-empty` fence 块而非损坏 JSON |
| 前端 JSON.parse 失败 | try-catch 降级为 `<pre><code>` 普通代码块 |
| values 全零/null 渲染空图 | 后端检测并设置 `values: []` + `emptyMessage`；前端检测后展示空状态 |
| 流式输出中 JSON 不完整导致多次无意义解析 | 仅围栏闭合后才触发 Vue 组件挂载，markdown.js fence 回调天然保证 |
| 饼图标签重叠 | 后端限制 ≤6 slice + 前端环形图 + `avoidLabelOverlap` + `minMargin` |
| 百分比 vs 绝对值着色规则不同 | `metricType` 字段驱动；percentage 用固定阈值，absolute 用动态相对阈值 |
| LLM 自由生成的 Mermaid 不受控制 | 保留 Mermaid 通道，chart-json 仅为后端固定 6 场景使用 |
| 两个 ECharts 通道并存 | chart-json 通道独立，不与 table 解析通道耦合 |
| 懒加载导致首帧无图 | `defineAsyncComponent` 加载时间极短（vendor-echarts 已预构建），且有 Loading 占位 |

---

## 七、非目标

- **不替换 AnalysisChart.vue**：从 HTML table 解析数据的通道继续存在
- **不删除 Mermaid**：流程图、时序图、甘特图仍是 Mermaid 的最佳场景
- **不改变 API 契约**：后端仍输出 Markdown，仅 fence 语言从 `mermaid` 变为 `chart-json`
- **不增加新 npm 依赖**：ECharts 已在项目依赖中；Jackson 已在后端依赖中

## 八、迁移后效果对比

| 维度 | 迁移前（Mermaid） | 迁移后（ECharts） |
|------|-----------------|-----------------|
| 柱状图颜色 | CSS nth-of-type 轮询，无数据含义 | 百分比固定阈值 / 绝对数值动态色阶，回调函数逐点着色 |
| 交互 | 无 | Tooltip 悬停显示精确数值 |
| 坐标轴 | Mermaid 内部布局，label 易重叠 | ECharts 网格系统，自适应宽度，智能截断 |
| 饼图 | 静态扇区，标签易重叠 | 环形图 + `avoidLabelOverlap` + `minMargin`，最多 6 扇区 |
| 平均线 | Mermaid line 系列，样式不可控 | ECharts markLine，虚线 + 标签 |
| 空数据 | 可能渲染空坐标轴白图 | 明确空状态展示 + 文字说明 |
| 加载态 | 骨架屏 + 白屏闪烁 | 流式 Loading 占位 → ECharts 渐进动画 |
| JSON 解析失败 | — | 降级为普通代码块，不白屏 |
| ECharts 加载时机 | — | `defineAsyncComponent` 懒加载，无图表不加载 |
| 渲染性能 | SVG DOM 操作较慢 | Canvas 渲染，大数据量下更快 |

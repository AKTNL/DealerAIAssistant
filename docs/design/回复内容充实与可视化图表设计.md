# 回复内容充实与可视化图表 — 设计方案

## 背景

当前模型回复结构统一但信息量偏弱，结论只有 1 句话、数据支撑只有 2 列简表、分析建议混排偏通用，视觉上只有表格比较单调。

## 设计目标

1. **内容充实**：Conclusion 扩展为 2-3 条要点（含优先级和经营判断）、Data Support 保留表格并新增图表、Short Analysis 拆分为数据归因与可执行建议
2. **可视化**：引入 Mermaid 图表作为表格的视觉补充，每个场景按数据特点选择合适的图表类型
3. **中文段落标题**：中文回答的四个段落标题中文化

## 段落标题对照

| 原标题 | 中文 |
|--------|------|
| `## Conclusion` | `## 核心结论` |
| `## Data Support` | `## 数据支撑` |
| `## Short Analysis` | `## 经营分析` |
| `FOLLOW_UP_QUESTIONS:` | `追问：` |

英文回答保持原标题不变。

## 一、内容充实

### 1.1 规则引擎侧（RuleBasedAnalyticsService.java）

**核心结论段落改造**：从 1 句话扩展为 2-3 条 Markdown 列表要点，包含：
- 核心发现（最值得关注的数字/门店）
- 优先级判断（哪些门店/指标需要立即关注）
- 经营影响（该发现对整体业务意味着什么）

**数据支撑段落改造**：保留 HTML 表格（全量数据），表格下方追加 Mermaid 图表代码块。图表仅展示 Top 5 或异动明显的样本，表格保留全量数据——避免柱状图因门店过多而拥挤。

**经营分析段落改造**：拆分为两段，使用结构化模板：

- **数据归因**（2-3 条）：`[指标变化] + [具体对象] + [可能原因]`
  - 例："A门店达成率仅 45.2%，较平均值低 27.6 个百分点，推测与其在途商机转化周期偏长有关"
- **可执行建议**（2-3 条）：`[动作] + [对象] + [预期目标]`
  - 例："立即清理 A门店的 12 条逾期跟进任务，预期一周内将达成率提升至 55% 以上"

**追问**：保持 2 个，内容更贴合当前分析结果。

### 1.2 外部模型侧（PromptFactory.java）

`buildGroundedModelPrompt` 和 `buildConversationModelPrompt` 中：
- 去掉 "concise" 类过度约束
- 明确要求 Conclusion 写 2-4 条要点
- 要求在 Data Support 中包含 HTML table 和 Mermaid 图表
- 要求 Short Analysis 引用具体数字
- 中英文段落标题要求同步更新

## 二、6 场景 Mermaid 图表设计

### 2.1 场景与图表类型

| 场景 | 图表类型 | Mermaid 类型 | 展示内容 |
|------|---------|-------------|---------|
| 目标达成分析 | 水平柱状图 + 平均线 | `xychart-beta` | Top 5 / Bottom 5 门店达成率对比，标注平均值线 |
| 商机漏斗分析 | 饼图 | `pie` | 商机阶段分布 |
| 销售跟进分析 | 柱状图 | `xychart-beta` | Top 5 门店未完成 vs 逾期任务数 |
| 活动效果分析 | 水平柱状图 | `xychart-beta` | 各活动达成率排序（Top 5） |
| 线索来源分析 | 饼图 | `pie` | 各来源线索量占比、各来源转化率 |
| 门店对标分析 | 柱状图 | `xychart-beta` | 门店达成率对比（Top 5 + Bottom 5） |

图表仅展示 Top 5 或异动明显的样本，HTML 表格保留全量数据。表格给精确数值，图表给一眼可见的对比。

### 2.2 Mermaid Label 安全清洗

门店名称、活动名称、车型名称若包含括号 `()` 或特殊符号，会导致 Mermaid 解析失败。生成 Mermaid 代码前，对所有 Label 执行清洗：

```
清洗规则：
- 移除括号及括号内内容：name.replaceAll("（[^）]*）", "").replaceAll("\\([^)]*\\)", "")
- 移除 Mermaid 保留字符：" ' { } [ ] 替换为空格或去除
- 长度截断：超过 10 个字符截断并加 "…"
- 兜底：清洗后为空则用 "门店-N" / "活动-N" 编号替代
```

### 2.3 xychart-beta 降级策略

`xychart-beta` 在部分 Mermaid 版本中可能存在渲染位置偏移。降级链路：

1. 优先使用 `xychart-beta`
2. 若渲染异常，后端同一方法同时生成一份备用文本条形图（Progress Bar 风格），在 Mermaid 代码块后以 HTML 注释形式嵌入
3. 前端检测到 `xychart-beta` 渲染失败时，展示条状图兜底

条状图兜底格式（纯 HTML + CSS，不依赖 Mermaid）：
```
▌▌▌▌▌▌▌▌▌▌ A门店  45.2%
▌▌▌▌▌▌▌▌▌▌▌▌▌▌ B门店  68.5%
▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌ C门店  72.0%
```

## 三、前端 Mermaid 集成

### 3.1 新增依赖

```bash
npm install mermaid
```

### 3.2 Mermaid 初始化与主题配置

在 `AssistantMessage.vue` 中初始化 Mermaid，配色方案匹配前端 UI 色系（当前 style.css 变量）：

```
UI 色系参考：
- 主色（文字/强调）：#111111
- 背景白：#ffffff
- 浅灰背景：#f3f3ef
- 边框浅灰：#d9d9d4
- 警示红：#8f3f35
- 页面底色：#f8f8f6
```

```js
mermaid.initialize({
  startOnLoad: false,
  theme: 'base',
  themeVariables: {
    primaryColor: '#f3f3ef',
    primaryTextColor: '#111111',
    primaryBorderColor: '#d9d9d4',
    lineColor: '#c8c8c2',
    secondaryColor: '#ffffff',
    tertiaryColor: '#8f3f35',
    fontSize: '13px'
  }
});
```

### 3.3 CSS 防跳变

`.mermaid-chart` 容器在 Mermaid 渲染完成前设置最小高度和淡入动画，消除 done 后短暂样式跳变：

```css
.mermaid-chart {
  min-height: 180px;
  opacity: 0;
  transition: opacity 0.3s ease;
}

.mermaid-chart.mermaid-rendered {
  opacity: 1;
}
```

渲染完成后 JS 追加 `.mermaid-rendered` class 触发淡入。

### 3.4 markdown.js 改动

在自定义 `fence` 规则中增加 Mermaid 分支：

- 当 `langName === "mermaid"` 时输出 `<pre class="mermaid-chart">原始内容</pre>`
- 不经过 highlight.js 处理
- 保留原始 Mermaid 文本内容

### 3.5 AssistantMessage.vue 改动

- 引入 `mermaid`
- 流式过程中 `mermaid-chart` 容器显示为普通代码块样式（无渲染）
- 收到 `done` 事件后调用 `mermaid.run({ querySelector: '.mermaid-chart' })` 批量渲染
- 渲染完成后追加 `.mermaid-rendered` class 触发淡入
- 渲染失败：捕获异常，移除 `mermaid-chart` class 改为展示条状图兜底内容

### 3.6 流式兼容

- SSE 流式传输期间不触发 Mermaid 渲染
- 仅在流式完成（`done` 事件）后批量渲染
- 配合 `useChat.js` 的 done 回调触发

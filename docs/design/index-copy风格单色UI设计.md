# Index Copy 风格单色 UI 设计

## 目标
重构当前黑白前端方向，使页面结构和布局节奏部分参考 `index copy.html`，同时保留已批准的单色编辑视觉体系。

## 设计意图
- 借鉴 `index copy.html` 的结构优势。
- 不借用其青/蓝色调、渐变、发光或玻璃态效果。
- 保持最终 UI 以白色为主、黑色主导、灰色分层。
- 让产品更像一个专注的分析工作空间，而非演示原型。

## 参考边界

### 借鉴
- 集中的登录入口流程。
- 更纤细的产品头部。
- 更清晰的左侧边栏分组节奏。
- 更紧凑、更像产品的聊天工作区布局。
- 消息内容与输入控件之间更强的分隔。

### 不借鉴
- 青色或蓝色强调色。
- 渐变条、发光或玻璃效果。
- 柔和的 SaaS 风格圆角和装饰性悬停效果。
- 带有彩色强调的演示风格状态标签。

## 视觉体系
- 页面画布：`#f8f8f6`
- 主表面：`#ffffff`
- 主文字：`#111111`
- 次要文字：`#5f5f5f`
- 辅助文字：`#8a8a8a`
- 边框：`#d9d9d4`
- 更强边框：`#c8c8c2`
- 阴影保持极简且结构性，不营造氛围感。
- 圆角保持紧凑，主要为 `8px` 到 `12px`。
- 按钮保持扁平单色。

## 页面行为

### 登录页
- 脱离当前的分割编辑舞台风格。
- 使用页面中央的单一聚焦入口区域。
- 保留 Logo 在标题上方，但视觉权重低于大标题。
- 以干净垂直的节奏堆叠 Logo、标题、简短描述、输入框和操作按钮。
- 使登录交互保持窄而直接，更像一个受控的入口点而非落地卡片。
- 保持底部注释低调且低对比度。

### 聊天页
- 保持侧边栏 + 主内容区的应用框架。
- 使头部更薄、更像产品，更接近参考结构。
- 使侧边栏读起来像问题组和会话上下文的支持轨道。
- 使主聊天区域更像文档画布而非浮动卡片。
- 通过灰度表面、边框和布局权重区分用户消息和助手消息。
- 将输入区域固定为工作区底部清晰的控制栏。

## 组件规则
- 主按钮：黑色背景，白色文字。
- 次要按钮：白色背景，黑色文字，灰色边框。
- 输入框：白色背景，细灰色边框，聚焦时呈微妙深色环。
- 侧边栏分区：清晰分组、紧凑间距、极少装饰。
- 顶部头部：纤细、克制、横向产品标识。
- 消息块：内容优先，更少药丸感，更像文档。
- 用户消息块可携带比助手块更强的灰色表面。
- 链接、标签和元数据保持灰度，除非用户后续批准专用强调色。

## 实施范围
- 重做 `LoginView.vue` 和 `frontend/src/style.css`，使登录页变为聚焦的单一入口构图，而非分割的双面板舞台。
- 在 `frontend/src/style.css` 中重做聊天工作区间距和外壳样式，更好地匹配参考结构。
- 保持当前 Vue 组件拆分完整，除非某文件确实难以干净地设置样式。
- 复用此分支中已有的测试/工具基础。
- 当布局钩子或登录结构发生实质性变化时更新测试。

## 非目标
- 不回归彩色品牌。
- 不克隆 `index copy.html`。
- 不更改后端或数据流。
- 不做前端外壳和表现层之外的无关重构。

## 风险
- 过度借鉴参考可能使产品感觉像演示而非工具。
- 借鉴太少可能使当前 UI 感觉像只是换了颜色而非结构改进。
- 如果不谨慎处理间距和字号层级，登录页简化可能意外降低视觉层次。

## 验证
- 登录页呈现为聚焦的单一入口屏幕。
- 聊天页明显更像产品工作空间，而非浮动卡片的落地布局。
- 最终 UI 保持单色，避免蓝/青强调色。
- 侧边栏、顶栏、消息区和输入区域在结构上更接近 `index copy.html`。
- UI 在桌面端和移动端断点下保持可读和稳定。

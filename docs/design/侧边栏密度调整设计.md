# 侧边栏密度调整设计

**日期：** 2026-05-13

## 目标

在可折叠题库升级后收紧左侧边栏的视觉密度。

此优化应：

- 移除"新对话"入口周围的外部白色卡片修饰
- 将"新对话"图标和标签向左边缘靠近
- 不再将"新对话"按钮居中为全宽核心元素
- 使六个侧边栏模块在垂直方向感觉更紧凑

## 范围

本设计仅涵盖侧边栏表现层变更。

包含：

- "新对话"操作容器样式
- "新对话"按钮对齐和宽度行为
- 模块卡片间距和内边距
- 展开的问题列表间距

排除：

- 聊天提交行为
- 手风琴交互行为
- 本地化问题内容
- 移动端打开/关闭行为
- 顶栏或主聊天布局

## 问题陈述

当前侧边栏仍保留了早期工作流卡片处理中过于宽松的间距：

- "新对话"操作位于白色框架块内，赋予其不必要的视觉权重
- "新对话"按钮横跨整行，因此图标和标签读起来是居中的，而非类似工具栏
- 六个模块有充足的内边距和间距，使侧边栏感觉比必要的高和松散

用户希望获得更紧凑、更靠左锚定的侧边栏，而不改变侧边栏的行为方式。

## 设计决策

### 1. 从"新对话"行移除卡片包裹外观

"新对话"操作应保持其现有按钮样式，但其父容器不应再渲染为白色盒装分区。

这意味着：

- 在 `.sidebar-action-block` 上不做带边框的卡片修饰
- 按钮后不做分区式背景
- 更小的周围内边距使按钮更靠近侧边栏导轨

### 2. 左锚定"新对话"按钮

按钮不应再读起来像居中、全宽的 CTA。

变更：

- 移除强制全宽布局
- 保持图标和标签水平对齐
- 保持内容左对齐
- 让按钮宽度适应其内容或适度最小宽度，而非拉伸到边缘

这保留了视觉强调同时使其感觉像实用操作而非英雄块。

### 3. 压缩侧边栏模块间距

六个题库模块应通过减少以下三处垂直节奏变得更紧凑：

- 分区内边距
- 模块卡片内边距
- 模块间间距

模块仍应保持可读且明显可点击。

### 4. 略微压缩展开问题间距

展开的问题按钮应保持其当前样式和直接发送行为，但问题堆叠应占用更少的垂直空间。

变更：

- 略微缩小问题按钮内边距
- 略微缩小问题间距
- 略微收紧模块标题和问题列表之间的顶部间距

这是密度调整，不是重新设计样式。

## 组件和样式影响

### `frontend/src/components/layout/ExampleSidebar.vue`

预期无结构性变更。

当前标记已支持预期调整：

- `.sidebar-action-block`
- `.primary-sidebar-button`
- `.sidebar-module-card`
- `.sidebar-module-questions`
- `.sidebar-module-question`

### `frontend/src/style.css`

此文件应吸收全部变更。

预期选择器更新：

- `.sidebar-action-block`
- `.primary-sidebar-button`
- `.sidebar-section`
- `.sidebar-module-card`
- `.sidebar-module-card + .sidebar-module-card`
- `.sidebar-module-questions`
- `.sidebar-module-question`

## 视觉方向

结果应感觉：

- 更紧凑
- 更编辑化
- 更少盒装感
- 更左对齐和实用驱动

不应感觉像重新设计。应感觉像相同侧边栏移除了过多内边距。

## 测试策略

因为这是主要的样式变更，逻辑测试应保持稳定。

验证应确认：

- 现有侧边栏组件测试仍然通过
- 工作区框架测试仍然通过
- 不引入组件行为变更

目标验证：

- `src/components/__tests__/ExampleSidebar.spec.js`
- `src/components/__tests__/WorkspaceChrome.spec.js`

## 验收标准

本设计在以下条件满足时完成：

- "新对话"行不再有外部白色框架块
- "新对话"按钮内容在视觉上更靠近左边缘
- 按钮不再作为全宽核心元素拉伸
- 六个模块总体上占用更少的垂直空间
- 展开的问题列表略微更紧凑而不改变行为

## 一、启动初始化链路

```
Sample Data.xlsx                     H2 内存数据库
       │                                   ▲
       │  ExcelImportService               │
       │  (Apache POI 解析)                 │ JPA Repository
       │                                   │
       ▼                                   │
┌──────────────┐    JPA saveAll    ┌───────┴───────┐
│ 解析 Sheet:   │─────────────────▶│  内存表:       │
│ Campaign     │                  │  campaign      │
│ Opportunity  │                  │  opportunity   │
│ Lead         │                  │  lead          │
│ Task         │                  │  task          │
│ Target       │                  │  target        │
│ AE Target    │                  │  dealer (派生) │
└──────────────┘                  └───────────────┘
```

**步骤说明：**
1. Spring Boot 启动 → `ExcelImportService` 读取 `Sample Data.xlsx`
2. 逐 Sheet 解析为 Entity 对象 (JPA `@Entity`)
3. 通过 `JpaRepository.saveAll()` 批量写入 H2 内存表
4. 基于 Target / Opportunity / Lead 数据构建派生 `Dealer` 表
5. Campaign ↔ Opportunity 之间补齐关联门店信息
6. Task ↔ Opportunity 之间补齐关联门店信息
7. 初始化完成，所有数据 API 就绪

## 二、用户认证链路

```
浏览器                    前端 (Vue)                  后端 (Spring)             Session
  │                         │                             │                       │
  │ ① 打开页面               │                             │                       │
  │ ──────▶                 │                             │                       │
  │                         │ ② 检查 sessionStorage       │                       │
  │                         │   auth 标记不存在            │                       │
  │  ◀── 展示登录遮罩层 ◀──  │                             │                       │
  │                         │                             │                       │
  │ ③ 输入口令，点击提交     │                             │                       │
  │ ──────▶                 │ ④ POST /api/auth/verify     │                       │
  │                         │   { "key": "****" }         │                       │
  │                         │ ────────▶                   │                       │
  │                         │                             │ ⑤ 比对 app.auth.access-key
  │                         │                             │                       │
  │                         │ ⑥ { "success": true }       │                       │
  │                         │ ◀────────                   │                       │
  │                         │                             │                       │
  │                         │ ⑦ 写入 sessionStorage       │                       │
  │                         │   auth: "true"              │                       │
  │  ◀── 展示主页面 ◀─────── │                             │                       │
```

**关键数据：**
- `sessionStorage.auth` → 登录标记（关闭浏览器自动清除）
- `localStorage.locale` → 语言选择（zh/en，持久化）
- `localStorage.brand_session_id` → Spring AI 会话 ID（持久化）

## 三、AI 对话流 (核心链路)

```
浏览器 (Vue)                  后端 (Spring)                  OpenAI 兼容模型
  │                              │                               │
  │ ① 用户输入问题并发送          │                               │
  │ ──────▶                      │                               │
  │                              │                               │
  │ ② 创建 UserMessage (UI)      │                               │
  │ ③ 创建 AssistantMessage 占位  │                               │
  │ ④ POST /api/chat/stream      │                               │
  │   { sessionId, message }     │                               │
  │ ────────▶                    │                               │
  │                              │                               │
  │                              │ ⑤ ChatService.streamChat()     │
  │                              │   构造 System Prompt           │
  │                              │   + 历史对话 (多轮上下文)       │
  │                              │   + 用户当前问题               │
  │                              │                               │
  │                              │ ⑥ Spring AI ChatClient        │
  │                              │   含 Tool Definition           │
  │                              │ ──────────────────────────────▶│
  │                              │                               │
  │                              │     (AI 可能调用 Function Tool) │
  │                              │ ◀── Tool Call Request ────────│
  │                              │     (Dealer/Opportunity/...)   │
  │                              │ ──── Tool Result ────────────▶│
  │                              │                               │
  │                              │ ⑦ SSE 事件流返回               │
  │                              │ ◀──────────────────────────────│
  │                              │                               │
  │  ◀── event: progress ◀────── │  (工具调用进度，如"正在查询商机数据")  │
  │  ◀── event: message  ◀────── │  (正文分片，逐步追加)            │
  │  ◀── event: message  ◀────── │  (更多分片...)                  │
  │  ◀── event: done     ◀────── │  (data: [DONE])                │
  │                              │                               │
  │ ⑧ 前端 useSseParser 解析      │                               │
  │   按 event/data 行处理        │                               │
  │   追加到 assistant 消息       │                               │
  │                              │                               │
  │ ⑨ 响应结束                    │                               │
  │   解析 FOLLOW_UP_QUESTIONS    │                               │
  │   解析 <think> 思考过程       │                               │
  │   渲染 Markdown + 追问按钮    │                               │
```

**SSE 事件协议 (Spring AI 链路)：**

| 事件类型 | data 内容 | 前端行为 |
|----------|-----------|----------|
| `progress` | 工具调用进度文本 (JSON) | 追加到 assistant 消息 steps 列表 |
| `message` | 正文分片 (JSON 字符串) | 追加到 assistant 消息正文 |
| `done` | `[DONE]` | 结束流式接收，进入最终渲染 |

**AI 输出约束：**
- 优先给结论 → 数据支撑和分析 → Markdown 组织
- 末尾必须附带 `FOLLOW_UP_QUESTIONS:` 区块 (2个追问)
- 不暴露内部工具名或接口名
- 语言跟随用户提问语言

## 四、数据查询 API 链路

```
浏览器 (Vue) / AI Tool              后端 (Spring)              H2 数据库
  │                                     │                         │

  │ ?dealerCode=xxx&startDate=xxx       │                         │
  │ ──────────────────▶                 │                         │
  │                                     │                         │
  │                     X-API-Key 校验   │                         │
  │                           │         │                         │
  │                           ▼         │                         │
  │                      ApiKeyFilter   │                         │
  │                           │         │                         │
  │                     校验通过 ▼      │                         │
  │                                     │                         │
  │                                     │ DataQueryService        │
  │                                     │ → JPA Specification      │
  │                                     │ ────────────▶           │
  │                                     │                         │
  │                                     │           JpaRepository │
  │                                     │           findAll()     │
  │                                     │ ◀────────────           │
  │                                     │                         │
  │ ◀── JSON 分页响应 ◀──────────────── │                         │

```

**6 类数据查询接口：** `/api/v1/data/{dealers|opportunities|campaigns|tasks|targets|leads}`

## 六、会话清理链路

```
浏览器 (Vue)                  后端 (Spring)
  │                              │
  │ DELETE /api/chat/{sessionId} │
  │ ────────────────▶           │
  │                              │ ① InMemoryChatMemory.clear(sessionId)
  │                              │ ② 释放该会话的对话历史
  │                              │
  │ ◀── 200 OK ◀─────────────── │
  │                              │
  │ ③ 前端生成新 sessionId       │
  │   存入 localStorage          │
  │                              │
  │ ④ 清空消息列表                │
  │   显示"会话已清空"提示        │
```

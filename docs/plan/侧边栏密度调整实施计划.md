# 侧边栏密度调整实施计划

> **面向代理工作器：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐步实施此计划。步骤使用复选框（`- [ ]`）语法进行追踪。

**目标：** 通过移除外部框架化的"新对话"块、左锚定"新对话"按钮以及压缩模块/问题间距来收紧左侧边栏布局，而不改变侧边栏行为。

**架构：** 保持 `ExampleSidebar.vue` 中的侧边栏 DOM 和交互模型不变。几乎完全在 `frontend/src/style.css` 中应用优化，仅使用轻量回归断言来证明组件形态仍然完整，且没有行为变更泄漏到工作区框架。

**技术栈：** Vue 3、Vitest、Vue Test Utils、`frontend/src/style.css` 中的共享 CSS

---

## 文件结构

### 需修改的文件

- `frontend/src/style.css`
  - 从 `.sidebar-action-block` 移除盒装修饰
  - 左锚定 `.primary-sidebar-button`
  - 减少侧边栏分区/模块/问题间距
- `frontend/src/components/__tests__/ExampleSidebar.spec.js`
  - 仅当需要支持聚焦样式的变更时才添加或调整轻量结构断言

### 工作中需检查的文件

- `frontend/src/components/layout/ExampleSidebar.vue`
  - 确认现有标记已暴露所需选择器，不需要结构化编辑
- `frontend/src/components/__tests__/WorkspaceChrome.spec.js`
  - 确认密度调整后不需要预期变更
- `frontend/package.json`
  - 使用 `npm.cmd run test -- ...` 进行聚焦验证

# 侧边栏流程入口实施计划

> **面向agentic工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实施此计划。步骤使用复选框（`- [ ]`）语法进行跟踪。

**目标：** 将聊天侧边栏升级为带有工具提示、可展开步骤预览和可直接发送的侧边栏问题的流程卡片，并复用现有的聊天提交流程。

**架构：** 保持后端和聊天流式传输流程不变。将更丰富的侧边栏场景元数据移入专用前端数据文件，通过更结构化的 `ExampleSidebar` 渲染该数据，并将侧边栏问题点击直接连接到 `useChat` 的 `submitPrompt(promptOverride)`。

**技术栈：** Vue 3、Vitest、Vue Test Utils、Vite、纯CSS。

---

## 文件映射
- `frontend/src/constants/sidebarFlows.js`：管理 `zh` 和 `en` 两种语言的六个侧边栏流程定义，包括摘要、工具、步骤、逻辑提示和问题文本。
- `frontend/src/i18n/messages.js`：导入流程定义并通过 `promptModules` 为两种语言环境暴露它们。
- `frontend/src/components/layout/ExampleSidebar.vue`：渲染流程卡片、卡片展开、工具标签、步骤预览和可直接发送的问题按钮。
- `frontend/src/views/ChatView.vue`：将仅填充的侧边栏行为替换为通过 `submitPrompt(question)` 的直接提交。
- `frontend/src/style.css`：将旧的折叠面板样式替换为与当前工作区外壳匹配的流程卡片样式。
- `frontend/src/__tests__/sidebarFlows.spec.js`：验证已发布侧边栏流程数据的数据契约。
- `frontend/src/components/__tests__/ExampleSidebar.spec.js`：验证流程卡片渲染、展开和直接发送事件。
- `frontend/src/components/__tests__/WorkspaceChrome.spec.js`：验证 `ChatView` 将侧边栏问题点击转发到 `submitPrompt(question)` 并仍然关闭移动端侧边栏。

### 3.3 关键技术决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| AI 链路框架 | Spring AI 1.0 | 与 Spring Boot 深度集成，原生 Function Calling 支持 |
| 数据库 | H2 内存库 | POC 不要求持久化，启动快速，零配置 |
| 数据初始化 | Apache POI + JPA | 需求明确要求 Excel → 内存库，POI 是 Java 生态标准方案 |
| 会话管理 | InMemoryChatMemory | 按 sessionId 存储多轮历史，POC 不要求持久化 |
| API Key 过滤 | 自定义 Filter + 白名单路径 | 简单够用，POC 不引入 Spring Security |
| 鉴权 | 单一口令 + sessionStorage | POC 不要求用户体系 |

## 四、AI 链路架构 (Spring AI)

```
ChatService.streamChat()
    │
    ├── 1. LanguageDetector.detectLanguage(message)
    │      → 判断用户输入语言 (zh/en)
    │
    ├── 2. PromptFactory.buildSystemPrompt(locale)
    │      → 构建 System Message: 角色设定 + 输出约束 + 数据背景
    │
    ├── 3. SessionMemoryService.getHistory(sessionId)
    │      → 获取多轮对话历史 (User/Assistant 消息对)
    │
    ├── 4. Spring AI ChatClient
    │      .prompt()
    │      .system(systemPrompt)
    │      .messages(historyMessages)
    │      .user(currentMessage)
    │      .tools(dealerTools, opportunityTools, campaignTools,
    │             taskTools, targetTools, leadTools, currentDateTools)
    │      .stream()
    │      → 发送到 OpenAI 兼容 API
    │
    ├── 5. 中间处理:
    │      - AI 决定是否调用 Function Tool
    │      - Tool 执行 JPA 查询返回数据
    │      - 数据注入 AI 上下文继续生成
    │
    └── 6. SSE 输出:
           event: progress → 工具调用进度
           event: message  → 正文分片
           event: done     → [DONE]
```

## 五、部署与启动

### 5.1 开发模式

```
终端 1: cd backend && ./mvnw spring-boot:run     → localhost:8081
终端 2: cd frontend && npm run dev                → localhost:5173
```

### 5.2 生产打包

```
cd backend && ./mvnw clean package       # Maven 自动执行 npm ci + npm run build
java -jar backend/target/agent-poc-backend-0.0.1-SNAPSHOT.jar
```

前端构建产物 (Vite build) 自动输出到 `backend/src/main/resources/static/`，由 Spring Boot 作为静态资源服务。

### 5.3 配置项

```yaml
server.port              → SERVER_PORT            (默认 8081)
app.auth.access-key      → APP_ACCESS_KEY         (默认空，需显式配置)
app.security.api-key     → APP_API_KEY            (默认空，需显式配置)
app.excel.path           → APP_EXCEL_PATH         (默认 classpath:Sample Data.xlsx)
spring.ai.openai.api-key → SPRING_AI_OPENAI_API_KEY
spring.ai.openai.base-url → SPRING_AI_OPENAI_BASE_URL
```

## 六、接口汇总

### 6.1 鉴权接口 (无需 API Key)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/verify` | 访问口令校验 |

### 6.2 聊天接口 (无需 API Key)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat` | 同步聊天 |
| POST | `/api/chat/stream` | SSE 流式聊天 |
| DELETE | `/api/chat/{sessionId}` | 清空会话记忆 |

### 6.3 数据查询接口 (需要 X-API-Key)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/data/dealers` | 门店查询 |
| GET | `/api/v1/data/opportunities` | 商机查询 |
| GET | `/api/v1/data/campaigns` | 活动查询 |
| GET | `/api/v1/data/tasks` | 任务查询 |
| GET | `/api/v1/data/targets` | 目标查询 |
| GET | `/api/v1/data/leads` | 线索查询 |

### 6.4 API Key 白名单 (不校验)

- `/api/auth/**`
- `/api/chat/**`
- `/swagger-ui/**`
- `/v3/api-docs/**`
- `/h2-console/**`

# 业务架构 — 某豪华汽车品牌经销商 AI 分析助手 (Agent POC)

> 来源：Agent-POC-需求文档 v2026-04-29  
> 说明：梳理业务域、分析场景、用户角色与业务价值。

---

## 一、业务定位

本项目是某豪华汽车品牌**经销商 AI 分析助手**的 POC（概念验证），目标是在短周期内构建一套可运行、可演示的 Web 系统，让管理/销售管理人员通过自然语言与 AI 对话，快速获取经销商销售与市场数据的分析报告。

**核心价值：** 以对话式 AI 替代传统 BI 报表查询，降低经销商数据分析门槛。

**非生产系统：** POC 使用本地样例数据，不做真实系统对接。

## 二、业务主题域 (6 类一级分析场景)

```
┌──────────────────────────────────────────────────────┐
│              经销商 AI 分析助手 (6大业务主题)           │
├──────────────┬──────────────┬────────────────────────┤
│  销售分析     │  市场分析     │  经营分析               │
├──────────────┼──────────────┼────────────────────────┤
│ 1.目标达成    │ 4.市场活动   │ 5.经营对标              │
│ 2.商机漏斗    │              │                        │
│ 3.销售跟进    │              │ 6.线索来源与趋势         │
└──────────────┴──────────────┴────────────────────────┘
```

| 编号 | 业务主题 | 核心分析内容 | 数据来源 | 典型问题举例 |
|------|----------|-------------|----------|-------------|
| S1 | 目标达成分析 | 各经销商/门店的销售目标完成率、同比/环比趋势 | Target + Dealer | "北京地区上个月的目标达成情况如何？" |
| S2 | 商机漏斗与转化分析 | 商机各阶段的转化率、漏斗瓶颈、在途商机分析 | Opportunity + Dealer | "各阶段的商机转化率是多少？哪个环节流失最大？" |
| S3 | 销售跟进分析 | 销售任务的完成活跃度、跟进及时性 | Task + Opportunity + Dealer | "本周销售跟进任务完成率最低的三个门店是哪些？" |
| S4 | 市场活动规划与效果分析 | 活动类型分布、活动投入产出比、新客获取效果 | Campaign + Opportunity + Dealer | "上周市场活动带来的线索转化率如何？" |
| S5 | 经营对标分析 | 跨门店、跨区域的关键指标横向比较 | 多表联合 | "表现前 5 和后 5 的门店分别是谁？差距在哪些指标？" |
| S6 | 线索来源与自然流量趋势 | 线索来源分布、趋势变化、线索转化效果 | Lead + Dealer | "线索来源占比如何？自然流量趋势是上升还是下降？" |

## 三、业务数据域

```
                        ┌──────────┐
                        │  Dealer  │  ← 派生实体（从 Target / Opportunity / Lead 构建）
                        │  (门店)   │
                        └────┬─────┘
           ┌─────────────────┼─────────────────┐
           ▼                 ▼                  ▼
    ┌──────────┐     ┌──────────────┐     ┌──────────┐
    │  Target  │     │ Opportunity  │     │   Lead   │
    │  (目标)   │     │   (商机)      │     │  (线索)   │
    └──────────┘     └──────┬───────┘     └──────────┘
                            │
                   ┌────────┴────────┐
                   ▼                 ▼
            ┌──────────┐     ┌──────────┐
            │ Campaign │     │   Task   │
            │ (市场活动) │     │ (跟进任务) │
            └──────────┘     └──────────┘
```

**实体关系说明：**
- `Dealer` 是核心维度实体，关联所有业务实体
- `Campaign` ↔ `Opportunity`：一次市场活动产生多个商机
- `Task` ↔ `Opportunity`：商机产生销售跟进任务
- `Target` ↔ `Dealer`：门店维度的销售目标
- `Lead` ↔ `Dealer`：门店来源的销售线索

## 四、用户角色

| 角色 | 典型用户 | 使用方式 | 关注点 |
|------|----------|----------|--------|
| 演示人员 | 内部演示/汇报人员 | 预设问题演示 AI 分析能力 | 流畅的用户体验、清晰的分析结论 |
| 经销商管理人员 | 某豪华汽车品牌区域销售经理 | 自然语言输入业务问题 | 快速获得数据洞察、支持决策 |
| 评估人员 | 研发/产品/业务同学 | 评估 POC 效果、验证链路 | 功能完整性、AI 回复质量、可扩展性 |

## 六、POC 范围边界

### 范围内 ✅
- 单页 Web 应用
- 基于口令的简单访问控制
- 单聊天工作区 (仅 Spring AI 链路)
- SSE 流式 AI 回复
- 6 类业务数据查询
- AI 多轮对话 + Markdown 报告 + 追问建议

### 范围外 ❌
- 用户体系 / RBAC / SSO
- 真实 CRM / DMS / CDP 对接
- 数据权限隔离
- 持久化聊天记录
- 生产级安全治理
- 移动端 App
- 中文/英文之外的国际化语言

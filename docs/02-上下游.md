## 二、上游依赖

### 2.1 外部服务依赖

| 上游系统 | 用途 | 协议 | 方向 | 备注 |
|----------|------|------|------|------|
| OpenAI 兼容模型 API | AI 对话能力（Chat Completion） | HTTPS / SSE | 后端调用 | 可配置 base-url，支持任意兼容平台 |
| H2 内存数据库 | 内置数据存储 | JDBC | 内嵌 | 不依赖外部数据库，启动时自动初始化 |

### 2.2 数据来源

| 上游数据 | 用途 | 格式 | 加载方式 |
|----------|------|------|----------|
| `Sample Data.xlsx` | 样例业务数据（Campaign / Opportunity / Lead / Task / Target / Dealer） | Excel (.xlsx) | 服务启动时 Apache POI 读取并初始化到 H2 |

### 2.3 技术栈上游 (编译/运行时)

| 上游组件 | 版本 | 说明 |
|----------|------|------|
| JDK | 21 | Java 运行时 |
| Spring Boot | 3.4.5 | 应用框架 |
| Spring AI | 1.0.0 | AI 集成框架 |
| H2 Database | (managed) | 内存数据库 |
| Apache POI | 5.4.1 | Excel 解析 |
| highlight.js | ^11.11.1 | 前端代码高亮 |
| markdown-it | ^14.1.1 | 前端 Markdown 渲染 |
| mermaid | ^11.15.0 | 前端图表渲染 |
| Vite | ^6.3.5 | 前端构建工具 |
| Vue | ^3.5.13 | 前端框架 |

## 三、下游产出

### 3.1 面向用户

| 下游产出 | 载体 | 消费者 | 说明 |
|----------|------|--------|------|
| AI 分析回答 | Web 页面 (聊天区) | 经销商管理人员、演示人员、内部评估人员 | Markdown 结构化报告：结论 + 数据支撑 + 分析 + 追问建议 |
| 流式打字体验 | Web 页面 (SSE) | 同上 | 实时推送进度与正文分片 |
| 数据查询报表 | JSON API 响应 | 前端 / AI 工具层 | 6 类数据查询接口 |
| 前端静态资源 | 浏览器加载 | 终端用户 | Vite 构建产物嵌入后端 static 目录 |

### 3.2 面向开发/运维

| 下游产出 | 格式 | 消费者 | 说明 |
|----------|------|--------|------|
| 启动说明 | 文档 | 开发 / 评审人员 | 本地一键启动步骤 |
| 配置说明 | 文档 | 开发 / 评审人员 | application.yml 配置项与环境变量覆盖 |
| 演示问题集 | 文档 / 聊天 | 演示 / 评审人员 | 一组预设的经销商分析问题 |
| H2 Console | Web UI (`/h2-console`) | 开发人员 | 运行时查看内存数据 |
| Swagger UI | Web UI (`/swagger-ui`) | 开发 / 联调人员 | API 接口文档 |
| Actuator 端点 | HTTP (`/actuator/health,info`) | 运维 / 监控 | 健康检查信息 |

## 四、外部关联系统 (POC 范围内不对接，但需预留认知)

| 系统 | 关系 | POC 阶段处理 |
|------|------|-------------|
| CRM (客户关系管理) | 商机 / 线索真实来源 | 用 Sample Data.xlsx 模拟 |
| DMS (经销商管理系统) | 门店 / 销售真实来源 | 用 Excel Dealer Sheet 模拟 |
| CDP (客户数据平台) | 客户数据整合 | 不涉及 |
| SSO / LDAP | 统一身份认证 | 用简单口令替代 |
| API Gateway | 统一入口与限流 | 不涉及 |
| 监控告警系统 | 生产运维 | 不涉及 |
